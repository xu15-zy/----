# 敦煌六大核心纹样 · 网站优化审计与建议

> 审计对象：Next.js 14 + React Three Fiber 项目（app / components / data / public）
> 审计维度：交互、内容、排版、页面美观、性能、无障碍
> 结论先行：设计语言已成型且统一，问题集中在「移动端不可用」「3D/动画资产未落地」「开发占位文案外泄」「无障碍与性能细节」。下面按优先级列出。

---

## 一、已经做得好的地方（先肯定）

- **配色系统自洽**：石青 / 赭石 / 米白 / 朱砂 / 金 / 墨 六色命名体系清晰，且全部带暖色调，不踩「纯灰纯黑」反模式。
- **字体选择合理**：Noto Serif SC + Noto Sans SC，避开了 Arial/Inter/system-ui，且全站仅 2 个字族，符合规范。
- **组件原语统一**：`.card-dh / .btn-dh / .btn-dh-ghost / .divider-dh / .pill-dh` 一致复用，间距走 Tailwind 4px 基准，无「13px/22px」式随意边距。
- **真实交互**：3D 拖拽旋转、三层嵌套显隐重组、滚动揭示，都不是摆设。
- **动效克制**：无 bounce/elastic，时长落在 0.7s~3s 区间，`prefers-reduced-motion` 已覆盖 `.reveal`。
- **语义化与 SEO**：`lang="zh-CN"`、`metadata`、动态 `generateMetadata` 齐全。

---

## 二、P0 · 上线前必须修（影响可用性 / 呈现）

### 1. 移动端完全没有导航 ⛔
`components/Nav.tsx` 中导航 `<ul>` 和「进入纹样」按钮都标了 `hidden md:flex` / `hidden md:inline-flex`。
**后果**：<768px 设备上用户只能看到 Logo，无法跳转到历史/体系/纹样/传承任意页面，等于移动端「死站」。
**建议**：加一个移动端菜单（抽屉或下拉）。Nav 已是 `"use client"`，直接补状态即可：

```tsx
"use client";
import { useState } from "react";
// ... 在 header 内、logo 之后加：
const [open, setOpen] = useState(false);
<button
  className="md:hidden rounded-lg border border-jin/40 px-3 py-1.5 text-mo/70"
  aria-label="打开菜单" aria-expanded={open} onClick={() => setOpen(v => !v)}
>{open ? "✕" : "☰"}</button>
{open && (
  <ul className="md:hidden absolute left-0 right-0 top-16 border-b border-jin/30 bg-mibai/95 backdrop-blur">
    {links.map(l => (
      <li key={l.href}>
        <Link href={l.href} onClick={() => setOpen(false)}
          className="block px-5 py-3 text-sm text-mo/80 hover:bg-shiqing/5">{l.label}</Link>
      </li>
    ))}
  </ul>
)}
```

### 2. 六纹样中 4 个 3D 模型根本没接上 ⛔
`data/patterns.ts` 里六纹样 `model` 指向 `/models/rendong.glb` 等，但 `public/models/` 实际只有 **`qianfo.glb`、`yun.glb`**。
根目录虽有 `莲花.glb / 忍冬.glb / 波纹.glb` 等，但它们**不在 `public/models/` 下**，且文件名不匹配（`lianhua`≠`莲花`、`santu` 连 glb 都没有）。
**后果**：忍冬/莲花/水波/三兔 四个详情页永远显示「3D 模型占位」，核心卖点塌掉。
**建议**：
- 把 4 个缺失模型按接口命名放入 `public/models/`：`rendong.glb / lianhua.glb / shuibo.glb / santu.glb`（santu 需另出 glb 或把 `.mp4` 改为视频播放）。
- 根目录那一堆 `*-副本.glb / *.svg` 是源文件，应移到 `assets/` 或 `src-assets/`，别和 `public/` 混放，避免打包误入。

### 3. 「差异动画」全是 SVG 占位，未落地 ⛔
`components/PatternAnimation.tsx` 用内联 SVG 画出各纹样示意，角标写着「差异动画占位（待替换 MP4）」，而 `public/animations/` 目录为空。
**后果**：页面反复强调的「差异化动画输出」目前是几何占位，与文案承诺不符。
**建议**：二选一——(a) 接受 SVG 动画为正式版，把角标占位文案删掉；或 (b) 真正接入 `public/animations/*.mp4`（`<video autoPlay muted loop playsInline>` 替换 SVG）。若短期做不完，至少把「待替换 MP4」字样移除，避免显得未完成。

### 4. 开发占位文案直接出现在线上 UI ⛔
多处外泄开发状态：`3D 模型占位（待补充 .glb）`、`差异动画占位（待替换 MP4）`、`动画占位`、Footer「占位开发版」。
**建议**：正式交付前统一清理；占位态改用中性文案（如「模型加载中 / 敬请期待」），且 Footer 去掉「占位开发版」。

## 三、P1 · 交互与无障碍

### 5. `prefers-reduced-motion` 未覆盖循环动画
`.reveal` 已处理，但 `dh-ripple / dh-dot / dh-drift / dh-orbit / dh-grow / animate-slow-spin` 仍在无限运行。
**建议**：在 `globals.css` 补：

```css
@media (prefers-reduced-motion: reduce) {
  .dh-ripple, .dh-dot, .dh-drift, .dh-orbit, .dh-grow, .animate-slow-spin {
    animation: none !important;
  }
}
```

### 6. 键盘焦点样式缺失
`.btn-dh / .btn-dh-ghost / 卡片链接` 仅有 `transition`，无 `focus-visible` 描边。键盘用户难以定位焦点。
**建议**：在 `globals.css` 加统一焦点环（用品牌强调色）：

```css
a:focus-visible, button:focus-visible {
  outline: 2px solid #c9a227;
  outline-offset: 2px;
  border-radius: 0.5rem;
}
```

### 7. 图层显隐按钮缺 `aria-pressed`
`LayerRecombine` 的显隐 toggle 是 `<button>` 但无 `aria-pressed` / `aria-label`，屏幕阅读器读不出状态。
**建议**：`aria-pressed={show[key]}` 并给按钮加 `aria-label={`切换${row.label}`}`。

### 8. Canvas 加载期空白闪烁
`ModelViewer` 内 `<Suspense fallback={null}>`，模型加载完成前画布区域是空白；外层的「加载三维视图…」只在 `mounted/exists` 判断前出现，Suspense 阶段无兜底。
**建议**：把 Suspense fallback 换成骨架屏（与外层一致的米白占位 + 文字），避免突兀空白。

## 四、P2 · 性能

### 9. 字体用 CSS `@import` 渲染阻塞
`globals.css` 顶部 `@import url("...fonts.googleapis.com...")` 是渲染阻塞写法。
**注意**：CJK 字体（Noto Serif/Sans SC）用 `next/font/google` 有子集限制（不含中文子集），不建议强行 next/font。
**建议**：保留 Google Fonts，但改为在 `app/layout.tsx` 的 `<head>` 里用 `<link rel="preconnect">` + `<link rel="stylesheet" ...display=swap>` 非阻塞加载，并删掉 CSS 里的 `@import`。或把 woff2 自托管到 `public/fonts/`。

### 10. three.js 未懒加载
每个详情页都静态引入 R3F + three（约 600KB+），即便模型缺失也会拉起整个 3D 运行时。
**建议**：用 `next/dynamic(() => import("@/components/ModelViewer"), { ssr:false, loading: () => <骨架/> })`，且 `ssr:false` 必须在客户端组件内调用——可新建一个 `ModelSection.tsx`（`"use client"`）包裹 dynamic 引入。

### 11. `Environment preset="sunset"` 有网络依赖
drei 的 preset 会从外部 CDN 拉 HDR，离线/防火墙环境会失败、光照变黑。
**建议**：改用本地 `<Environment files="/env/xxx.hdr">` 或纯 `<lightformer>`/多光源方案，去掉外部依赖。

## 五、P3 · 排版与页面美观

### 12. 字号缺乏系统比例
全站字号是 `text-4xl/6xl/3xl/2xl/xl/lg/sm/xs` + `text-[11px]` 拼凑，没有显式 modular scale（1.25 或 1.333）。
**建议**：在 `tailwind.config.ts` 里定义一套语义化字号 token（如 `display/title/heading/body/small/caption`），全站引用，保证层级节奏一致。

### 13. Hero CTA 暗卡偏平
首页底部 `bg-mo p-10` 暗卡信息密度低、无装饰，略显空。
**建议**：叠一层极淡的纹样 SVG 水印（低透明度金色描边），或加左右两枚「忍冬/莲花」角饰，提升「敦煌感」而不抢内容。

### 14. `SectionHeading` 强制居中
`components/SectionHeading.tsx` 写死 `text-center`，详情页等场景左对齐阅读更顺。
**建议**：加 `align?: "center" | "left"` 参数，默认 `center`，详情页传 `left`。

### 15. 中文 label 字距过宽
`tracking-[0.4em]` 用在中文「敦煌 · 佛教装饰符号体系」上，字间隙偏大显松散。
**建议**：中文标签用 `tracking-[0.2em]` 即可；英文大写标签（NARRATIVE / SIX PATTERNS）保留 `0.3~0.4em`。

### 16. 缺 favicon 与社交分享图
无 `favicon.ico`、`icon`、`apple-icon`，也无 Open Graph 图。
**建议**：放一枚「金线缠枝」或「三兔」SVG favicon 到 `app/icon.svg`（Next 原生支持），并加 `openGraph` 字段。

## 六、P4 · 内容

### 17. 「内:中:外 = 1:3:1」比例未解释
详情页 Hero 与「三层比例」卡片展示比例，但访客不懂数字含义（是面积比？半径比？视觉权重？）。
**建议**：在比例旁加一行注脚，如「按视觉权重近似占比」，或配一个迷你三层环形示意图。

### 18. `public/images/` 17 张图从未使用
这些图未在任何页面以 `<img>` 引用，疑似死资产或待接入。
**建议**：要么在「历史/文化传承」页作为壁画实景插图接入（能显著提升真实感与美观度），要么移出 `public/` 减重。

### 19. `system` 页硬编码色值
`app/system/page.tsx` 写死 `accent="#b23a2e"`，应复用 `accentHex.zhusha` 保持单一数据源。
**建议**：`import { accentHex } from "@/data/patterns"`，改 `${accentHex.zhusha}`。

## 七、建议落地顺序

| 优先级 | 项 | 工作量 |
|--------|----|--------|
| P0 | 移动端导航 (#1) | 小 |
| P0 | 补齐/重命名 4 个 3D 模型 (#2) | 中（依赖资产） |
| P0 | 动画占位落地或去文案 (#3,#4) | 小 |
| P1 | reduced-motion / focus-visible / aria (#5,6,7,8) | 小 |
| P2 | next/dynamic 懒加载 + 字体非阻塞 (#9,10,11) | 中 |
| P3 | 字号体系 / 暗卡装饰 / favicon (#12~16) | 中 |
| P4 | 比例注脚 / 图片接入 / 去硬编码 (#17~19) | 小 |

> 一句话总结：**设计骨架很正，先把「移动端能逛、模型能转、占位文案清掉」三件事做完，站点就能从课程作业质感跃升到可对外展示的成品。**
